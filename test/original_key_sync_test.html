🎸 左側原調同步功能驗證指南

修正說明

修正後的功能會讓左側的「原調」欄位隨著 AG (吉他實際彈奏調性) 同步變動。

邏輯： 左側原調顯示 = AG 調性 = 目標調性 - Capo

測試場景

場景 1: 基本 Capo 變更

初始狀態： Key: Bb (AG:G) Capo: 3
左側原調顯示：G
操作： 將 Capo 從 3 變更為 0
期望結果：
• 預覽區：Key: Bb (AG:Bb) Capo: 0
• 左側原調顯示：Bb ← 應該同步變更
• 左下方目標調顯示：Bb (保持不變)

場景 2: Capo 增加

初始狀態： Key: Bb (AG:Bb) Capo: 0
左側原調顯示：Bb
操作： 將 Capo 從 0 變更為 3
期望結果：
• 預覽區：Key: Bb (AG:G) Capo: 3
• 左側原調顯示：G ← 應該同步變更
• 左下方目標調顯示：Bb (保持不變)

場景 3: 目標調性變更

初始狀態： Key: Bb (AG:G) Capo: 3
左側原調顯示：G
操作： 將目標調性從 Bb 變更為 C
期望結果：
• 預覽區：Key: C (AG:A) Capo: 3
• 左側原調顯示：A ← 應該同步變更
• 左下方目標調顯示：C (跟隨變更)

手動測試步驟

  1. 打開主應用程式 (index.html)
  2. 觀察左側原調欄位的初始值
  3. 點擊不同的 Capo 按鈕 (0, 1, 2, 3...)
  4. 確認左側原調欄位會隨著 AG 調性同步變更
  5. 點擊不同的目標調性按鈕
  6. 確認左側原調欄位也會相應調整
注意： 左側原調應該始終與預覽區 Key 行中的 AG 調性保持一致

計算公式驗證

公式： 左側原調 = 目標調性 - Capo
範例：
• 目標調性 Bb (= 10), Capo 3 → 原調 = (10-3+12)%12 = 7 = G ✓
• 目標調性 Bb (= 10), Capo 0 → 原調 = (10-0+12)%12 = 10 = Bb ✓
• 目標調性 C (= 0), Capo 3 → 原調 = (0-3+12)%12 = 9 = A ✓
